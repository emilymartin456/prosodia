# 设计笔记：韵律与表现力控制

这份笔记记录几个不那么显然的设计决定，方便日后回看。

## 为什么把情感建成「相对增量」

每种情感（`Emotion`）存的是**相对中性**的韵律增量：音高偏移（半音）、音高范围倍率、
语速倍率、能量倍率。好处有二：

1. **可按强度线性插值**：`intensity=0` 时任何情感都退化为中性，`intensity=1` 是标称强度，
   中间平滑过渡。实现上就是 `1 + (delta - 1) * k`。
2. **可叠加手动覆盖**：用户的 `rate` / `pitch_shift` 通过 `StyleVector.combine`（倍率相乘、
   偏移相加）叠在情感之上，互不打架。

最终所有控制汇聚成一个 `ProsodyTargets`，并在 `clamped()` 里裁剪到安全区间，避免极端值把
合成器带崩。

## 声调如何变成基频曲线

`f0.py` 用几个「相对半音控制点」描述每个普通话声调（受五度标记启发）：

| 声调 | 形状（半音） | 说明 |
|------|--------------|------|
| 1 阴平 | (4, 4)      | 高平 |
| 2 阳平 | (-1, 5)     | 上升 |
| 3 上声 | (-2, -4, 0) | 低降后微升 |
| 4 去声 | (5, -3)     | 高降 |
| 5 轻声 | (0, -1)     | 短而弱 |

控制点插值到目标帧数，乘以 `pitch_range`、加上 `pitch_shift`，再由半音换算回赫兹。
这样音高范围（情感的重要维度）与整体音高解耦，可以独立调。

## 变调放在注音之后、规划之前

三声连读、「不 / 一」变调在 `g2p/sandhi.py` 里对整段音节就地修改。它必须在时长/基频规划
**之前**跑，因为变调会改变声调，进而改变基频曲线。变调的「不 / 一」是按拼音表面形式识别的，
这是有意的简化：极少数同形词会误触发，但连续文本里绝大多数时候是对的。

## 韵律短语：规则够用，但留了接口

`RuleBasedPhraser` 只做两件事：尊重标点带来的边界，并在过长的无标点串里按音节预算插入短语
边界，让合成的语音「有地方换气」。它实现的是 `PhraseBreaker` 协议——将来换成模型预测的
短语边界，接口不变。

## 参考声码器的定位

`FormantSynthBackend` 是**参考实现**，不是要以假乱真。它用锯齿波激励 + FFT 域共振峰滤波做
元音，用整形噪声做辅音，全程确定性、可复现。它的价值是让整条管线**离线可跑、出声、可断言**，
从而把测试重心放在前端与韵律这些真正的难点上。追求音质时，实现 `AcousticBackend` 接入训练好的
声码器即可。
